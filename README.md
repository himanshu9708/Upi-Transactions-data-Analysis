# Upi-Transactions-data-Analysis
A data analysis project exploring UPI transaction trends using interactive dashboards. Includes insights on monthly balances, transaction amounts, city-wise activity, payment methods, device types, merchants, and user demographics to understand digital payment behavior.

## Dataset Used
 - <a href = "https://github.com/himanshu9708/Upi-Transactions-data-Analysis/blob/main/UPI%2BTransactions.xlsx">Upi+Transactions dataset</a>


 ## Project Objective
 The objective of this project is to analyze UPI transaction data to uncover meaningful patterns, trends, and insights related to digital payment behavior. By examining monthly transaction activity, remaining balances, city-wise performance, payment methods, device usage, merchant categories, and demographic factors, the goal is to understand how users interact with UPI across different segments. This analysis supports data-driven decision-making for improving financial services, enhancing user experience, identifying growth opportunities, and strengthening the overall digital payment ecosystem.

 # Questions(KPIs)
 
- Monthly Transaction Amount
- Monthly Remaining Balance
- City-wise Transaction Amount
- City-wise Remaining Balance
- Transactions by Payment Method
- Transactions by Device Type
- Transactions by Merchant Category
- User Demographics (Age Group & Gender Distribution)
- Transaction Volume by Purpose
- Transaction Count by Type (Send/Receive)

# Process

- Collected UPI transaction data including amount, balance, city, device, payment method, merchant, gender, and age group.
- Cleaned the dataset by removing duplicates, fixing missing values, and standardizing date and category formats.
- Transformed data by creating monthly groups, calculated fields, and KPI metrics for analysis.
- Modeled the data to support filtering by city, device type, payment method, gender, transaction type, and merchant.
- Built interactive dashboards with line charts, bar charts, and tables to visualize key trends and performance metrics.
- Analyzed insights to understand user behavior, city-level performance, payment preferences, and monthly transaction patterns.
- Documented findings and prepared visual summaries for easy interpretation and reporting

# Toola Used
- Power BI – For building interactive dashboards and visualizing transaction trends.
- Microsoft Excel – Used for initial data cleaning, formatting, and dataset preparation.
- Power Query – Applied for data transformation, merging, and shaping within Power BI.
- DAX (Data Analysis Expressions) – Used to create calculated columns, measures, and KPIs.
- CSV / Excel Datasets – Source files containing UPI transaction details.
- GitHub – For storing project files, documentation, and version control.
